O Git Pages é hoje minha solução preferida para hospedar aplicações web estáticas sem depender de infraestrutura complexa. O deploy vira um push, o custo segue zero para projetos open source e o conteúdo fica distribuído pertinho de quem acessa.

Atualmente tudo vive no GitHub Pages, mas já estou preparando um plano de disaster recovery replicando os mesmos artefatos no GitLab Pages. Quanto mais camadas, mais tranquilo fico para testar coisas diferentes no laboratório.

Para aproveitar essa abordagem é essencial que cada aplicação seja 100% client-side, usando as APIs modernas do navegador. Quando o projeto depende de backend dedicado eu compartilho em outra stack; o Git Pages entra como vitrine rápida.

## Sites principais
